SCENARIO: Bill - Load Client Context
CANONICAL SOURCE: Make Blueprint JSON (Bill - Load Client Context V2 blueprint (4).json)
WEBHOOK LABEL: Bill - Client ID Receiver V2
WEBHOOK URL: https://hook.eu2.make.com/4uq52ajluecic9p29n4dg3ypck6cgnxn
PURPOSE:
Load the full client context bundle for a given client_id.
Returns profile, plan, weeks, sessions, steps, nutrition,
history counts, engagement state, and contraindication placeholders.
Called at session start and whenever Bill needs a full context refresh.
Exercise Bests are NOT included — these are loaded directly by the
Flask backend from Google Sheets and cached separately.
Contraindications fields are returned as empty placeholders — they are
populated into the Clients sheet by the daily scheduled sync scenario,
not by this webhook.

-----------------------------
INPUT CONTRACT (JSON BODY)
-----------------------------
Flat JSON object. One key only:
- client_id (string) REQUIRED

No nesting. No other keys referenced.

Fragile bits / must-follow rules:
- filterRows searches Clients column A using case-insensitive text equal
- If client_id is missing the filter expression will fail and scenario errors
- client_id must exactly match the ID stored in the Clients sheet (case-insensitive)

-----------------------------
OUTPUT CONTRACT — NOT FOUND
-----------------------------
Response shape when client_id not found in Clients sheet:
{
  "status": "not_found",
  "client_id": "<echoed from input>"
}

Bill behavioural rule: if status is not_found, do not proceed with
any training or coaching actions. Inform the user their profile
does not exist and offer to start onboarding.

-----------------------------
BILL BEHAVIOURAL RULES
-----------------------------
- Always call this at the start of a new coaching session before
  any training, planning, or advice actions
- Never use cached context older than the current session without
  offering to refresh
- Exercise Bests are NOT in this payload — they are loaded and
  cached by the Flask backend directly from Google Sheets
- contraindications_temp and contraindications_chronic will always
  be empty in this response — read injuries and chronic_contraindications
  from client_profile instead (populated by daily sync)
- If plan_context.active_block is empty, no active training block exists —
  do not attempt to load weeks/sessions/steps; offer to create a new plan
